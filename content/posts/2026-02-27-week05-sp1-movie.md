This week was SP1, a group project building a movie repository backend using the TMDB API.

The task involved fetching Danish films from the TMDB API, storing them in PostgreSQL using JPA, and implementing DAO CRUD methods plus JPQL queries across four entities: Movie, Actor, Director, and Genre. Data was fetched via HTTP, parsed into DTOs, converted to entities in the service layer, and persisted through the DAOs.

Key challenges included handling duplicate entries from TMDB's own data, managing many-to-many relationships between movies and genres/actors, and ensuring the `overview` field used `TEXT` instead of `VARCHAR(255)` to handle longer descriptions.

Tests were written using JUnit 5 with TestContainers against a real PostgreSQL instance. All queries verified with clean test data before final submission.

**Entities:** Movie, Actor, Director, Genre. **External API:** TMDB `/discover/movie` + `/movie/{id}/credits`.
