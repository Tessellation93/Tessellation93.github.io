title: "Building in Public - A 12 Week Backend Journal"
---

This blog documents my exam project from start to finish. Every weekend, I'll be logging my progress, technical decisions, and debugging process as I build JavaJolt a RESTful Java API over the course of 12 weeks.

The goal isn't just to ship something. It's to build with intention, reflect on what's working and keep optimizing as I go.

JavaJolt started as an idea in my first semester, when I first began coding. Most resources for learning to code are either dry, dense or just exhausting to get through. A 30 page essay on loops isn't the most inspiring learning source. I wanted to build something that made the experience more engaging and more rewarding. A platform where learning Java actually feels worth showing up for, also a project with room for expansion.

On this project, I'm wearing every hat: tech lead, scrum master, and the entire development team. This journal is how I stay accountable to all three.
