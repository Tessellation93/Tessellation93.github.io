Week 01–02 — Project Setup, HibernateConfig & First Entity

6 February 2026·353 words·2 mins

Task
#

Week 1 - Choosing a project idea. Week 2 introduced JPA and Hibernate, the task was to get a real database connection running and persist at least one entity.

What was built
#

Set up the Maven project with the dk.javajolt package structure, configured Hibernate to talk to PostgreSQL, and created the first entity: Course.

Package layout from the start:

dk.javajolt/
  config/      ← HibernateConfig lives here
  entities/    ← JPA-mapped classes
  daos/        ← all database operations
  controllers/ ← HTTP handlers
  routes/      ← endpoint registration
  services/    ← business logic
  exceptions/  ← custom exceptions

HibernateConfig is a singleton. One EntityManagerFactory is created when the app starts and reused for the lifetime of the application. Individual EntityManager instances are created per operation and closed immediately after.

public class HibernateConfig {
    private static EntityManagerFactory emf;

    public static EntityManagerFactory getEntityManagerFactory() {
        if (emf == null) {
            emf = Persistence.createEntityManagerFactory("javajolt");
        }
        return emf;
    }
}

Course was the first entity because it sits at the top of the data hierarchy. Everything else: lessons, exercises, progress relate back to it.

@Entity
@Table(name = "courses")
@Getter @Setter @NoArgsConstructor
public class Course {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 200)
    private String name;

    @Enumerated(EnumType.STRING)
    @Column(nullable = false)
    private Difficulty difficulty;

    @Column(name = "is_deleted", nullable = false)
    private boolean isDeleted = false;

    @Column(name = "created_at", updatable = false)
    private LocalDateTime createdAt;

    @PrePersist
    protected void onCreate() {
        if (createdAt == null) createdAt = LocalDateTime.now();
    }

    public enum Difficulty { BEGINNER, INTERMEDIATE, ADVANCED }
}

Why?
#

Singleton EMF: An EntityManagerFactory is expensive to create, it scans all entities, sets up the connection pool and compiles JPA metadata. Creating a new one each individual request would be catastrophically slow. Singleton is the standard approach.

TEXT over VARCHAR for content fields: Lesson content and code examples can be long. VARCHAR(255) silently truncates. TEXT from the start avoids that bug entirely.

is_deleted from day one: Rather than deleting rows, a flag marks records as inactive. This preserves referential integrity if a user has progress records pointing to a course, hard-deleting that course breaks the relationship. Soft delete keeps everything consistent.
